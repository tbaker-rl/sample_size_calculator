A/B Test Calculator

Why do we need this calculator?

A/B testing is about separating signal (real behavior changes) from noise (random chance). If we test too few people, a "win" might just be luck. If we test too many, we waste time and traffic.

This tool uses statistics to find the "Goldilocks" number: the exact sample size required to scientifically prove that a change is real, not random.

Which metric type should I choose?

1. Proportions (Yes/No Data)

Use this when a user either does the action or doesn't. The data is binary.

  • Conversion Rate: Did they buy? (Yes/No)
  • Click-Through Rate: Did they click? (Yes/No)
  • Sign-up Rate: Did they register? (Yes/No)
2. Means (Averages/Continuous)

Use this when you are measuring "how much" or "how long". The data is a specific number for each user.

  • Average Order Value: How much did they spend? ($50, $120...)
  • Time on Page: How long did they stay? (30s, 90s...)
  • Pages per Session: How many pages did they view?

Choosing the wrong metric is the #1 reason A/B tests fail. Follow these 3 rules:

1. The "Proximity" Rule

Measure the immediate next step the user takes.

Example: If you change a Homepage Hero Image, measure Click-Through Rate, not Revenue. Revenue is too far away and has too much "noise" to detect a signal from an image change.

2. Quantity vs. Quality

Ideally, we want to test Revenue (Quality), but it often requires massive sample sizes.

Clicks (Quantity) are easier to test but might not lead to money.
Compromise: Test "Add to Carts" as a middle ground.
